- **B-2. YOLO-World/YOLOv12 파인튜닝** — 최고 정확도지만 라벨 데이터 + 학습 사이클 필요(별도 프로젝트). 도메인=실내라 실내 하자 라벨링 필요.

## 4. 오탐/미탐 신뢰성 프레임워크 (4가지 "모두 진행" 선택) — 설계

백엔드 중심이라 무인 배포 대신 **설계만**. 복귀 시 감독 하에 구현 권장.

### 4-1. 검수 워크플로 (사람 확정) — 일부 구현됨
- 이미: 카드에 확인/반려/오탐 버튼 + 등급(CONFIRMED/REVIEW/REFERENCE).
- 보완: **공식 보고·출장은 '점검자 확정(approved)' 건만**. 미확정은 '후보'로 명시 표기.
- 분쟁 방어: 책임 = 점검자 확정. (정책 + 보고서 생성 시 approved 필터)
- 코드: defectStore.applyReviewedDefect 이미 있음. 보고서 생성 경로에서 review_status='approved' 필터 추가.

### 4-2. 타입별 정밀/재현 분리
- 안전직결(균열·누수·단열): 미탐 최소 → 낮은 임계 + 다 잡고 REVIEW.
- 미관(오염·스크래치·코킹): 오탐 최소 → 높은 임계.
- 코드: test_stream.py `_ui_conf_gate` 이미 OOD-fragile 0.75 / insulation 0.30 분기 있음. 타입별 정책표로 확장.

### 4-3. 다중 프레임 교차검증 (temporal consistency)
- 여러 키프레임에 같은 하자(공간 IoU + 동일 클래스) 반복 → CONFIRMED 승격. 단일 프레임 → REVIEW.
- 일시적 오탐(한 프레임 반짝) 감소.
- 코드: test_stream.py `_video_inference_loop` 에 검출 누적 + IoU 매칭 후처리 추가. postprocess_config.yaml temporal_filter(window/min_detections) 정책 재사용 가능.

### 4-4. 근거 감사로그
- 모든 검출에 프레임 스냅샷 + 신뢰도 + VLM reasoning 기록 → 분쟁 시 증거.
- 일부 있음: broadcast 에 confidence, defect_source 포함. VLM reasoning(이미 hybrid_detector 가 생성)을 카드/DB 까지 전달.
- 코드: _broadcast_detection 에 reasoning 필드 추가 + DB defects 테이블 저장 + 프론트 카드에 표시.
